# Bootstrap New Project

## Когда использовать
- Owner просит развернуть обвязку / базу знаний / harness в проекте.
- Стартует новый проект (пустая папка или репозиторий без `CLAUDE.md` и `wiki/`).
- Нужно развернуть проверенную обвязку в свежем проекте независимо от стека.

## Что делает
Интерактивно спрашивает 4 вещи (стек, домен, деплой, база) и разворачивает в текущей папке:
- **`CLAUDE.md`** — правила проекта: session logging, секреты, структура знаний, 6 этапов, pre-push/post-deploy, multi-agent review.
- **`wiki/`** — структурное знание о коде (index, architecture, data-model, api, gaps, log, шаблон, подпапки models/controllers/services).
- **`docs/`** — процессное знание: `sessions/` (логи), `decisions/` (ADR), `audits/` (разборы), `operations/runbook-prod-deploy.md`, `reviews/review-format.md`, `gems.md`.
- **`docs/analytics/`** — статистика: летопись `runs.md`, сводка `dashboard.md` (считается скриптом), `incidents.md`.
- **`CODE-REVIEW.md`** — чек-лист проверки качества (растёт из разборов поломок).
- **`scripts/analytics/build_dashboard.py`** — детерминированный счётчик сводки.
- **`.github/`** — PR-template + CI-шаблон (dev auto / prod manual).
- **`.claude/`** — SessionStart hook (подгружает шапку wiki при старте сессии).

## Что НЕ делает
- Не выбирает стек за owner'а — спрашивает на старте.
- Не ставит авто-генерацию знаний через git-хуки (сознательно, см. ADR-0001 в шаблоне).
- Не подключает QMD/семантический поиск (не нужно до ~50 файлов в wiki).
- Не трогает существующий код проекта.

## Как запускается
Исполняй процедуру из `BOOTSTRAP.md` (в этой же папке skill'а) по шагам.
