## Предварительные условия
- [ ] CI на `main` зелёный (test + build + deploy-dev)
- [ ] Dev health OK
- [ ] Нет критических багов на dev
- [ ] Миграции прошли на dev без ошибок
- [ ] Owner проверил функциональность на dev (если применимо)

## Чек-лист перед деплоем
- [ ] Проверка dev: health + статус контейнеров/процессов + логи без ошибок
- [ ] Проверка текущего состояния prod (для возможного отката)
- [ ] **Backup БД перед prod** (если есть БД)

```bash
# пример (заполнить под стек/инфру)
# curl -sf https://dev.<host>/health && echo OK
# <backup command>
```

## Процедура деплоя
Ручной деплой оператором. CI на prod не ходит.
```bash
# 1. backup
# 2. при новых миграциях — применить
# 3. pull нового образа (после green CI build)
# 4. restart с новым образом
# 5. проверить статус + health
```

## Проверка после деплоя
- [ ] CI workflow зелёный
- [ ] Health check возвращает 200
- [ ] **Tail логи минимум 2 минуты** — зелёный CI ≠ работающий прод
- [ ] Функциональный smoke по критичным флоу
- [ ] Smoke на LLM/config — если они менялись

## Откат
```bash
# 1. остановить проблемные контейнеры/процессы
# 2. откатить на предыдущий образ/версию
# 3. при необходимости — откатить миграцию или восстановить из backup
# 4. проверить health
```

## Уведомления
- При успехе: отметить в session log, обновить version tag.
- При проблемах: откатить немедленно → уведомить owner'а → создать issue → задокументировать в session log + `docs/audits/`.
