# Gems — переносимые уроки и приёмы

Конденсат уроков из сессий и инцидентов + переиспользуемые приёмы. Растёт со временем. Кандидаты приходят из секций «Lessons» в session logs и из `docs/audits/`.

## Формат записи
```
### <название урока/приёма>
- **Контекст:** когда применимо
- **Урок/приём:** что делать (или чего не делать)
- **Источник:** session / audit / ссылка
```

## Записи

### Авто-генерация знаний через хуки даёт тихий дрейф
- **Контекст:** соблазн авто-обновлять wiki/доки через git-хуки.
- **Урок:** на старте веди знания **вручную в том же PR**, что меняет код. Авто-генерация (а) тихо расходится с кодом, (б) стоит денег на коммит, (в) не может быть source of truth. Автоматизируй точечно позже — только то, что устаревает быстрее всего.
- **Источник:** session {{DATE}}, ADR-0001.

### Investigate-first, fix-after
- **Контекст:** найден баг/инцидент, есть соблазн быстро предложить фикс.
- **Урок:** не предлагай фикс, пока трижды не проверил истинную причину. Probe-first для внешних систем (write-probe дешевле и точнее чтения логов). Неправильный фикс по первому предположению стоит часа reverse-engineering.
- **Источник:** методология insight-bot-mvp (handoff).
