# Статистика работы агента

Здесь живёт аналитика — в обычных текстовых файлах, по принципу **«летопись + сводка»**:

- **`runs.md`** — **летопись**. Сюда только дописывают строки, ничего не стирая. Одна строка = одна задача на одном этапе. Это история фактов — она не «протухает».
- **`dashboard.md`** — **сводка**. Итоговые таблички, посчитанные из летописи **скриптом** (не агентом). Помечена «посчитано автоматически, руками не править».
- **`incidents.md`** — поломки в бою (частота, ссылки на разборы в `docs/audits/`).

## Как заполнять летопись (`runs.md`)

Агент дописывает строку **на выходе с каждого этапа** работы. Колонки:

| Колонка | Что писать |
|---|---|
| `дата` | YYYY-MM-DD |
| `фича` | короткое имя задачи/фичи |
| `этап` | `требования` · `дизайн` · `код` · `тесты` · `релиз` · `бой` |
| `статус` | `одобрено` · `правки` · `пройдено` (для этапов без ревью) |
| `крит` | число критичных замечаний (0 если нет) |
| `важн` | число важных замечаний |
| `мелк` | число мелких замечаний |
| `покрытие` | % покрытия тестами (число, пусто если не мерили) |
| `время_мин` | сколько занял этап, минут (число, можно пусто) |
| `выпущено` | `да` · `нет` · `—` |
| `заметка` | свободный текст |

Числа пиши цифрами (для скрипта). Пустую ячейку оставляй пустой.

**Строки-образцы:** если имя фичи начинается с `demo-`, строка считается примером заполнения и **не учитывается** в сводке. Так в летописи можно держать наглядный образец, не искажая реальную статистику.

## Как пересобрать сводку

```bash
python3 scripts/analytics/build_dashboard.py
```

Скрипт читает `runs.md`, считает итоги и перезаписывает `dashboard.md`. Запускай вручную или перед отправкой кода. **Сам `dashboard.md` руками не правь** — изменения сотрёт следующая пересборка.

## Что считает сводка
- сколько задач прошло без правок с первого раза (% одобрено);
- средняя «тяжесть» замечаний (критичных на задачу);
- сколько времени уходит на каждый этап;
- текущее покрытие тестами;
- сколько выпущено;
- разбивка по этапам.
